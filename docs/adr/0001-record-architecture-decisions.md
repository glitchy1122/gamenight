# ADR-0001: Record architecture decisions

**Status:** accepted · **Date:** 2026-07-08

## Context
Solo project, long timeline, learning goals. Six months from now, "why did I pick X?"
will be unanswerable from code alone.

## Decision
Every non-obvious technical decision gets a one-page ADR in `docs/adr/`, using `template.md`.

## Consequences
Slight writing overhead per decision; in exchange, the reasoning survives, and
reversals are made against recorded context instead of vibes.
